Complex Types

MLine

Represents an assignment line. This is used to link invoice lines to lrn lines, invoice lines to invoice lines and discrepancies.

Child elements

Name Minimum Maximum Type Description
CompanyCode 1 1 typeCode

The company in which the MLine resides. This forms part of the key.
MLineId 1 1 typeDbGuid

The id of the assignment line.
Currency 1 1 CurrencyKey

The currency the MLine is currently in.
HomeCurrency 1 1 CurrencyKey

The home currency for the document
DualCurrency 0 1 CurrencyKey

The dual currency for the document
Group 0 1 GroupKey

The group the MLine is related to. It need not always have a group associated. If an invoice has been force posted (via the house keeping invoice process) then the MLine analysis will be performed on it but it won't be held within a group.

MatchableCreditNoteLineToInvoiceLineAssignment 0 1 typeBoolean

Indicates if this mline resulted from assignment of matchable invoice line to matchable credit note line.

Discrepancy 1 1 typePimMlineDiscrepancy

Indicates if the MLine represents a discrepancy or not. If it does then it describes the type of discrepancy handling to employ.

DiscrepancyType 0 1 typePimMlineDiscrepancyType

If the MLine does represent a discrepancy, is it a value, price or quantity discrepancy?
DiscrepancyLevel 0 1 typePimDiscrepancyLevel

If the MLine represents a discrepancy, is it a document or line level discrepancy?
DiscrepancyReasonId 0 1 typeDbGuid

The id of the reason for the discrepancy.

We need the three constituent parts of the discrepancy reason key because we can't refer to it via aggregation. This is because it is owned by the matching master.

DiscrepancyReasonCode 0 1 typeCodeB

The matching master code of the reason for the discrepancy.

We need the three constituent parts of the discrepancy reason key because we can't refer to it via aggregation. This is because it is owned by the matching master.

DiscrepancyReasonCompany 0 1 typeCodeB

The matching master company of the reason for the discrepancy.

We need the three constituent parts of the discrepancy reason key because we can't refer to it via aggregation. This is because it is owned by the matching master.

DiscrepancyReasonDescription 0 1 typeDescription

The description for the discrepancy.
DiscrepancyComment 0 1 typeTextB

The comment applied to this discrepancy.
DiscrepancyCommentOrigin 0 1 typePimDiscrepancyCommentOrigin

The origin of the comment, interactive or background.
ToleranceMaster 0 1 ToleranceMasterKey

The tolerance master used when applying the discrepancy.
RfcLine 0 1 LRNLineKey

If the MLine is of type RFC discrepancy then at some point an RFC will be generated. Each discrepancy MLine for the invoice will generate an RFC line. So once the discrepancies have been resolved each one will link to an RFC Line (which in practice is a negative LRN Line).

InvoiceLine 0 1 InvoiceLineKey

This is a key identifying the invoice line associated with this assignment line.
InvoiceLineCompanyCode 0 1 typeCode

The company code part of the invoice line key.

We need to do it this way to workaround the weakness in the domain model where it will try and load the invoice line as soon as it sees the MLine. This will fail sometimes as we might have created the invoice line as part of assignment and not assembled it yet.

InvoiceLineDocumentId 0 1 typeDbGuid

The document id part of the invoice line key.

We need to do it this way to workaround the weakness in the domain model where it will try and load the invoice line as soon as it sees the MLine. This will fail sometimes as we might have created the invoice line as part of assignment and not assembled it yet.

InvoiceLineLineNumber 0 1 typeLong

The line number part of the invoice line key.

We need to do it this way to workaround the weakness in the domain model where it will try and load the invoice line as soon as it sees the MLine. This will fail sometimes as we might have created the invoice line as part of assignment and not assembled it yet.

InvoiceLineDocumentNumber 0 1 typeDocNumAlpha

The document number of the invoice set on the MLine.
InvoiceLineDocumentCode 0 1 typeCodeB

The document master code of the invoice set on the MLine.
InvoiceLineValue 0 1 Money

The full value for the invoice line.
InvoiceLineQuantity 0 1 typeNumber

The full quantity for the invoice line.
InvoiceLineUnitPrice 0 1 Money

The unit price for the invoice line.
SupplierInvoiceNumber 0 1 typeText

The suppliers number for the invoice the MLine is assigned to.
LrnLine 0 1 LRNLine

The LRN line attached to the MLine.
SplitLine 1 1 typeBoolean

Determines whether the LRN line attached to the MLine has been split. If it has more than one MLine hanging off it then it has.

RelatedMLineId 0 1 typeDbGuid

The id of the MLine this relates to. We can only record the id and not a full proper reference because we will get a stack overflow error when the model tries to assemble the MLine as it is a circular reference.

DiscrepancyMLineIdList 1 1 MLineDiscrepancyMLineIdList

The id of any discrepancy MLines that have been generated for this particular one.

We can only record the id and not a full proper reference because we will get a stack overflow error when the model tries to assemble the MLine as it is a circular reference.

DiscrepancyParent 0 1 typeDbGuid

If this is a discrepancy MLine then this records the id of the MLine from which it was generated.

ValueOnly 1 1 typeBoolean

Returns true if the MLine records value only and not quantity.
Value 1 1 Money

The value this m line represents for the line. The currency of the value is that of the invoice currency for the group. The value can't exceed the total value of the line.

HomeValue 0 1 Money

The value of the m line in home currency.
DualValue 0 1 Money

The value of the m line in dual currency.
LrnValue 0 1 Money

The value of the mLine in the lrn lines currency.
DocumentRate 1 1 Rate

The effective rate on the document currency.
HomeRate 1 1 Rate

The effective rate on the home currency.
DualRate 0 1 Rate

The effective rate on the dual currency. Only present if company is configured with a dual currency.

ParentRate 0 1 Rate

The effective rate on the parent currency. Only present if required in order to convert between document, home and dual.

Quantity 0 1 typeNumberb

The quantity of the line that is involved in this particular match. The value can't exceed the quantity on the goods line. The value of all match analysis object quantities for the line must sum to the quantity held on the line.

This property only needs to be set if set on a quantity and price line.
UnitPrice 0 1 Money

The unit price for the mLine.

This property only needs to be set if set on a quantity and price line.
MLineTax 0 1 TaxMasterKey

The MLine can have one tax code.
Assigned 1 1 typeBoolean

Determines whether the m line has been used to assign two lines to each other. Will return true for the following situations:

  • The m line has an invoice and an lrn line set.
  • The m line has an invoice line and another m line set.

AssignedDate 0 1 typeHardb

MLineTaxValue 0 1 Money

ChangeDiscrepancyUser 0 1 UserKey

ChangeDiscrepancyComment 0 1 typeTextB

The comment added when change descrepancy is selected in workflow
ContractNumber 0 1 typeTextB

The contract number. Derived from the invoice line, if set, otherwise from derived from the LRN line, if set.

ReferenceCount 0 1 typeULongB The number of other objects in the system that reference this object.
TimeStamp 1 1 short The TimeStamp number of this MLine.

Group

The group object.

Child elements

Name Minimum Maximum Type Description
GroupId 1 1 typeDbGuid

Internal ID for the group, should never be shown to the user.
CompanyCode 0 1 typeCode

The company within which the group exists.
GroupReference 1 1 typeLong

An external reference for the users. This will be posted to financials for each Invoice. This will allow the browse function in financials to group all related invoices together.

MatchingMasterCode 0 1 typeCodeB

The matching master in use for the group. This doesn't always have to be set but in the main it will be. This can also be changed. If the user loads up a WIP group it will be set to that of the matching master for that session.

Only the master code will be stored. This is because when we are working with the group the user could have modified the matching master at runtime. We don't want to persist these changes to the database. So instead we will just record the code.

InvoiceList 1 1 InvoiceList

The list of invoices held within the group. During the life of a group there should always be at least one invoice in the group, until the group is committed. When the invoices are matched they are removed from the group, thus eventually there will be no invoices left in the group.

MLineList 1 1 MLineList

The MLines held within the group. There may be no MLines, this could happen if the group contains only invoices and no assignment has taken place.

When adding an LRN line to the group an MLine will need to be created to represent the portion to be added. The MLine is then placed in the group and hence that part of the LRN then grouped.

BalanceStatus 1 1 typePimGroupBalanceType

The balanced status of the invoices within the group. This must be calculated based on the following criteria:

  • If all of the invoice headers in the group are not balanced then the status is "Not balanced".
  • If some of the invoice headers in the group are balanced and some not balanced then the status is "Not balanced".
  • If all of the invoice headers in the group are balanced and then the status is "Fully Balanced".

AuthorisedStatus 1 1 typePimAuthorisedStatus

See the authorisation spec for the rules involved in calculating the authorisation status of the group.
