AnalysisLine

Represents an analysis line within the posting transaction.

Child elements

Name Minimum Maximum Type Description
LineNumber 1 1 typeLong

This is the line number of the analysis line.

The account code the analysis line is to be posted to. This MUST NOT include a destination code.

Destination 0 1 typeCodeB

The inter-company destination for this line.
LineSense 1 1 typeDebitCredit

The sense of the analysis line, i.e. debit or credit.

This is the gross value of the document being posted.

Quantity 0 1 typeNumberb

The quantity of items on the line. This can be blank if the line is specified as value only.

The second price of items on the line. Comes from the LRN.line

This is the gross home value of the document being posted. This property does not need to be set if the home currency is the same as that of the document.

PostingDocument

Represents a document that could be posted to financials. This is a closer representation of the transaction that is posted than the invoice.

Child elements

Name Minimum Maximum Type Description
FinancialsDocumentMaster 1 1 typeCode

The document master in financials to post the document to.

FinancialsDocumentNumber 0 1 typeDocNumAlphaB

The number to post the document to financials with.

FinancialsJournalDocument 0 1 typeCodeB

If the document was deferred matched then it will need to be reversed out at some point, this is the document to perform the reversing post on.

OriginalDocumentNumber 0 1 typeDocNumAlphaB

This will either be for a debit note, in which case it points a the invoice it is related to, or where this is a journal following suspense posting, in which case it points at the suspense posted doc.

OriginalDocumentCode 0 1 typeCode

This will either be for a debit note, in which case it points a the invoice it is related to, or where this is a journal following suspense posting, in which case it points at the suspense posted doc.

OriginalCompany 0 1 typeCode

This will either be for a debit note, in which case it points a the invoice it is related to, or where this is a journal following suspense posting, in which case it points at the suspense posted doc.

HoldPayment 0 1 typeBoolean

Should the payment be held? This will only be set to true if we have performed a suspense posting and the master has the hold payment option set.

ReleasePayment 0 1 typeBoolean

Should held payment be released?

YearPeriod 1 1 typeHPeriod

The year/period that the document belongs in.

MatchingReference 0 1 typeLong

The reference of the match the posting is involved in.

SummaryLine 0 1 SummaryLine

The summary line for the document.

The analysis lines for the document.

The tax lines for the document.
