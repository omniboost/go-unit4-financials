LRN

The LRN type has Document as its base type.

The logical receipt note.

Child elements

Name Minimum Maximum Type Description
DocumentCode 1 1 typeCode

The document code of the lrn in it's source system. We just store the code as provided to us and have no way of validating it is a proper document code. For POP it should be one of their document codes, for another procurement system it could just be a piece of text.

DocumentNumber 1 1 typePimLrnDocnum

The number of the lrn document. If the document has come from a procurement system (including POP) then this field should be populated for us. If the document is a system generated document, i.e. as a RFC, then we will need to create this number ourselves.

TraderCode 0 1 typeTraderCodeB

The trader code. Only used if the supplier code is an umbrella element
NetValue 1 1 Money

The value net of tax in document currency. Always need a value at the header level as it will be used to validate that the correct line values have been provided.

GrossValue 1 1 Money

The value gross of tax in document currency.
GrossHomeValue 1 1 Money

The value gross of tax in home currency.
GrossDualValue 0 1 Money

The value gross of tax in dual currency.
TaxValue 1 1 Money

The tax value in document currency.
TaxHomeValue 1 1 Money

The tax value in home currency.
TaxDualValue 0 1 Money

The tax value in dual currency.
AdviceNoteNumber 0 1 typePimAdviceNoteNumberb

The advice note number relating to the header.
DeliveryDate 1 1 typeHard

The date this LRN was delivered.
HeaderOrderNumber 0 1 typePimOrderNumberb

Derived from order details linked to LRN Lines. Where more than one order number then header value is null.

Invoice 0 1 InvoiceKey

Only applicable for RFC. The invoice this LRN relates to
SupplierInvoiceNumber 0 1 typePimSupplierInvoiceNumber

Only applicable for RFC. This can be derived from the invoice the RFC is linked to.

LineList 1 1 LRNLineList

The LRNs lines.
ExternalSystem 0 1 ProcurementSystemMasterKey

The external procurement system from where the LRN came from. The system will validate that the user extension master has the correct usage, i.e. the external procurement system one.

ExternalSystemDocumentKey 0 1 typeTextB

Field to record the key of the header in the external procurement system. PIM has no idea what procurement system this LRN has come from. Therefore it is upto the external system to make sure that it can uniquely identify where this header came from based upon the value held by this property. PIM will simply store whatever value is passed to it and use this when calling back to the external system.

ReceiptStatus 0 1 typePimLrnReceiptStatus

What stage is the LRN at, first or second stage? If the procurement system doesn't have two stage receiving set then it should set it directly to the second stage and set the second stage date to the date it was received.

SecondStageDate 0 1 typeHardb

When the LRN goes to the second stage this should record the date that happened.

MatchedStatus 1 1 typePimLrnMatchedStatus

The matched status of the LRN. The possible values are:

  • Not matched
  • Part matched
  • Fully matched

TermsCode 0 1 typeTermB

The terms code from the order
PrintableCommentList 1 1 LRNPrintableCommentList

The printable comment for the LRN header. At 11.2 this is only used to hold printable comments on RFC headers.

ExternalSystemLineKey 0 1 typeText

Field to record the key of the item line in the external procurement system. PIM has no idea what procurement system this LRN has come from. Therefore it is upto the external system to make sure that it can uniquely identify where this line came from based upon the value held by this property. PIM will simply store whatever value is passed to it and use this when calling back to the external system.

This field is non-mandatory because if we create an RFC it is an internal document and as such will have no external line key.

UnmatchedQuantity 0 1 typeNumber

The unmatched quantity on this LRN line. If the line is value only then this property should be set to null.

UnmatchedValue 1 1 Money

The unmatched value on this LRN line. If the line is value only then it is calculated from the difference between the value of the line and the sum of the mLines that are linked to matched invoices. If it has quantity and price then it is calculated from the unmatched quantity multiplied by the unit price.

MatchingCurrency 0 1 CurrencyKey

THIS PROPERTY IS FOR INTERNAL USE ONLY AND IS NEVER PERSISTED TO THE DATABASE, ALTHOUGH WE DO WANT A DATABASE COLUMN SO WE CAN HAVE A METADATA VOCAB FOR IT. Within assign and balance in interactive matching, this property is used soley by the client to allow the user to see the currency of the invoice to which this LRN is being assigned.

MatchingCurrencyValue 0 1 Money

THIS PROPERTY IS FOR INTERNAL USE ONLY AND IS NEVER PERSISTED TO THE DATABASE, ALTHOUGH WE DO WANT A DATABASE COLUMN SO WE CAN HAVE A METADATA VOCAB FOR IT. Within assign and balance in interactive matching, this property is used solely by the client to copy the converted currency value from an MLine into the LRNLine, so that the user can see lrn values in invoice currency.

The matched status of the lrn line. Possible values are:

  • Part matched - some but not all of the lrn line has been matched
  • Fully matched - all of the lrn line has been matched

Status to indicate whether the lrn line is in a wip group. If it is in 1 single group then the status will indicate the reason in the same way the groups wip status does. If it is in multiple wip groups it will use a status akin to #wip.
