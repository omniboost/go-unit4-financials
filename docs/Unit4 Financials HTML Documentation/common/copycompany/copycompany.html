GetMasterTypeProvidersRequest

The GetMasterTypeProvidersRequest type has Request as its base type.

GetMasterTypeProviders.

This function provides the caller with information about each of the master type providers known to the server. In order to service requests to this function the implementation must have access to the various product application servers. The application server may cache this information since it will only vary when a new product is added to the installation suite or a product is upgraded to a version that provides different master types. The information provided by this function will be enough for the client to display the CopyCompanyApp, the RangeEditApp. The common application server will require the XMLi connection details of each of the product application servers that are to provider input for copy company; the coda-common.properties file will include these details (in a manner similar to the coda-web.properties file contains details for the web server). The property definitions already exist so no extra code would be required to allow this. It does mean that the installer for the common server should be updated to ask for connection details for each of the copy company provider products.
