Security

Authorising access to Web Services

Access to individual Web services is controlled by Coda functional security, defined using Capability Master. When you call a web service you must supply a valid Coda user name and password.

Note

If the user does not have a default company, you can supply one in the logon credentials. To do this join the company name to the user name using a \ character. For example, if the user name is JOEUSER and the company name is HEADOFFICE, specify JOEUSER\HEADOFFICE.
