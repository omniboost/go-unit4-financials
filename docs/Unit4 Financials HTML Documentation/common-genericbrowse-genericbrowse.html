GenericBrowseService

Namespace: http://www.coda.com/common/schemas/genericbrowse/genericbrowse-13.0/webservice
WSDL: common-genericbrowse-genericbrowse.wsdl
Path: services/common/genericbrowse/genericbrowse-13.0

The following methods are available on this endpoint:

  • Print
  • Select
  • SelectForLedger
  • SelectForLinkMaintenance
  • SelectForTransactions

Print

Input Parameters

PrintRequest
Name Minimum Maximum Type Description
DataSecurityMasterCode 1 1 typeGeneralCode The code for the data security master that controls the data security that applies to the user. This is required to ensure that the user is not able to print or transmit lines that are filtered out when browsing.
RunTimeSelector 1 1 SelectorMaster Contains the selector master that was used to retrieve data, including any run-time changes the user made and any values the user entered in response to prompts. (This improves performance when finding the lines to print.)
OutputDeviceCode 1 1 typeGeneralCode The output device to use to print or transmit rows.
PrintFormatCode 1 1 typeGeneralCode Code of the print format to be used. The print format should .. a) if tables are being used, have its form type set to 'flow'. b) if the order of printing is important, have a sort order on it.
EmailTemplateCode 1 1 typeGeneralCode The code of an e-mail template master used to provide the subject, body and signature of generated emails.
ChosenRows 1 1 Selection

A selection containing the primary key cells of the rows that need to be printed. These should correspond to the primary key of the primary object of the dataview that this generic browse/print is based on. N.B. Printing only has the granularity to print dataview lines based on the primary key of the primary object of the dataview. Therefore if your generic browse screen displays two lines that both have the same primary key, it is not possible to print one without the other one also being printed. To get round this, define a new dataview that has a more suitable primary key.

DoLogging 1 1 boolean Specifies whether to write a log file. If this element is set to true, then the elements LogType and LogTitle must also be set.
LogTitle 1 1 string The title for the log file. This needs to be set only if the DoLogging element is set to true.
LogType 1 1 typeGenericOon The type of the log. This needs to be set only if the DoLogging element is set to true.

Return Value

This method returns no data.

Select

Input Parameters

SelectRequest
Name Minimum Maximum Type Description
PresenterCode 1 1 typeGeneralCode The code for the presenter master to be used to display the data.
SecurityCode 1 1 typeGeneralCode The code for the data security master to be used to control data security.
RunTimeSelector 1 1 SelectorMaster Contains the selector master to be used to retrieve data, including any run-time changes the user made and any values the user entered in response to prompts.
ExtraVocabs 0 1 VocabIdList A list of extra vocabulary items to return in the presentation results.

Return Value

SelectResponse
Name Minimum Maximum Type Description
Selection 1 1 Selection This element holds your data.
PrimaryKeyVocabs 1 1 VocabList

Contains vocabs detailing the primary key of the primary object (table) of the dataview that we are browsing. This is for the benefit of generic printing. N.B. This limits generic printing to only being able to print (or not print) documents based on the primary key of the dataview. e.g. if your presenter is showing you more than one line with the same primary key, you will not be able to print one but-not-the-others of these lines on an individual basis. You will only be able to print one 'document' containing all the line with the same primary key. Lines with other primary keys will appear in other 'documents'. To get round this limitation, create a new dataview with a primary key that suits your needs.
