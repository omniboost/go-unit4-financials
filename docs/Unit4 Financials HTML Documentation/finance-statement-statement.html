StatementService

Namespace: http://www.coda.com/efinance/schemas/statement/statement-14.0/webservice
WSDL: finance-statement-statement.wsdl
Path: services/finance/statement/statement-14.0

The following methods are available on this endpoint:

  • Browse
  • Generate
  • GetDetails
  • GetHeader
  • GetReferenceNumbers
  • Lock
  • Post
  • Unlock
  • Validate

Browse

Input Parameters

BrowseRequest
Name Minimum Maximum Type Description
SelType 1 1 DetailsLocation The source of the data you want to browse: Books, Intray or Archive.
GslCode 1 1 typeGeneralCodeB The code of the selector master you are using to select the statement lines.
WholeDocuments 1 1 boolean If TRUE, specifies that you want to browse whole documents.
RowLimit 0 1 typeULong The maximum number of lines you want to retrieve for browsing.
ApplicationFilter 1 1 VocListData The search criteria to be applied by the application when selecting the data. These are as set up in the selector master.
UserFilter 1 1 VocListData The additional search criteria specified by the user, to be applied when selecting the data. This may be, for example, a document code or a range of codes.
HierarchyFilter 0 1 VocListData The additional search criteria which will be applied to select data within specified hierarchies.
Columns 1 1 CellList The columns in which the retrieved data will be displayed.

Return Value

BrowseResponse
Name Minimum Maximum Type Description
Selection 0 1 Selection This element contains the data you have selected for browsing, which is returned from the server.

Generate

Input Parameters

GenerateRequest
Name Minimum Maximum Type Description
GetDetailsParams 1 1 GetDetailsParams The parameters for retrieving the statement lines for which you want to generate document proposals.
Params 1 1 GenerateParams Contains the parameters for generating documents for the statement lines.
LogID 1 1 int The name of the log to which the details of the Generate process will be written.

Return Value

GenerateResponse
Name Minimum Maximum Type Description
FullKey 1 1 FullKey The key containing the codes of the masters and the statement you want to use in Statement Processing.
DocProposals 0 1 DocProposals The generated documents containing the proposed transactions.
Folders 0 1 FolderList Contains the list of folders and information about the lines held in the folders.

GetDetailsRequest

Return Value

GetDetailsResponse

Details 0 1 Details Contains information about the statement lines and statement value.

GetReferenceNumbers

Input Parameters

GetReferenceNumbersRequest
Name Minimum Maximum Type Description
Key 1 1 Key The key containing the selection criteria for the Reference Number map.

Return Value

GetReferenceNumbersResponse
Name Minimum Maximum Type Description
Key 1 1 Key The key containing the selection criteria for the Reference Number map. This contains statement master codes and the statement reference numbers valid for each.
ReferenceNumberMap 0 1 ReferenceNumberMap The list of statement reference numbers mapped against statement masters, which you have received from the database.

Lock

Input Parameters

LockRequest
Name Minimum Maximum Type Description
Key 1 1 FullKey The key containing the code of the company, the masters and the statement, identifying the statement you want to lock for your own use in Statement Processing.

Return Value

LockResponse
Name Minimum Maximum Type Description
Key 1 1 FullKey The key containing the codes of the company, the masters and the statement, identifying the statement you want to lock for your own use in Statement Processing.
LockToken 0 1 typeText A random alpha-numeric string which is returned when you send a request to lock a statement. You need the LockToken if you want to initiate any further processing for the locked statement, as it indicates that you are the user with write-access to that table.

Validate

Input Parameters

ValidateRequest
Name Minimum Maximum Type Description
GetDetailsParams 1 1 GetDetailsParams The parameters for retrieving the statement lines you want to validate.
Folders 1 1 FolderList Contains information about the folders in which the statement lines are held.
LogID 1 1 int The name of the log to which the details of the Validate process will be written.

Return Value

ValidateResponse
Name Minimum Maximum Type Description
FullKey 1 1 FullKey The key containing the codes of the masters and the statement you want to use in Statement Processing.
Folders 1 1 FolderList The list of folders containing the statement lines, which have attempted to validate. The folders element for this response will be taken from the Folders element of ValidateRequest. The status values may be modified to indicate an error, or a change of posting location from books to Intray.
