ContractMasterLine

A line on the contract.

Child elements

Name Minimum Maximum Type Description
LineID 1 1 typeDbGuid

The unique identifier for the line.

LineNumber 1 1 typeULong

This controls the ordering of the lines in the contract, and is not part of the line's key.

Item 1 1 ItemKey

The key that identifies the item.

ItemDescription 1 1 typeText

The description of the item.

UnitOfMeasure 1 1 typeDescription

The Unit of Measure for the selected Item

StartDate 1 1 typeHard

The date from which this item can be purchased on the contract terms. Mandatory. Must be correct date format and less than or equal to End date and Review date, if either are set. Can be in the past.

EndDate 0 1 typeHardb

The date when purchase of this item on contract terms ends. Not mandatory. No entry treated as an ongoing arrangement i.e. never expires. If set, it must be in correct date format and greater-than-or-equal-to Start date and Review date, if set. Cannot be in the past when modified.

UnitPrice 0 1 Money

The price of a single item unit.

UnitPrice2 0 1 Money

The second price of a single item unit.

PriceMatrix 0 1 PriceMatrixMasterKey

The code of the price matrix master to be used instead of a unit price. Reference to a Price Matrix Master. Either a price matrix or unit price must be set, but not both. Must have the same currency as the Contract.

SupplierRef 0 1 typeDescription

The supplier's reference for the item.

Used to populate the item/supplier pricing details on the item master, when details are saved. Optional.

ProductName 0 1 typeDescription

The supplier's name for the item.

Used to populate the item/supplier pricing details on the item master, when details are saved. Optional.

MinQty 0 1 typeNumberb

The minimum quantity of the item, allowable under the contract.

FixedQty 0 1 typeNumberb

The fixed quantity which must be purchased under this contract.

QtyToDate 0 1 typeNumberb

The bought/ordered/requisitioned quantity of the item under the contract. Used to calculate the remaining quantity on a line with a fixed quantity.

DiscountValue 0 1 Money

The value of the discount in discount currency.

NetValueToDate 0 1 Money

The remaining net value of the item under the contract. The bought/ordered/requisitioned net value of the item under the contract. Used to calculate the remaining net value.

EditReason 0 1 typeDescription

A reason for editing this line. Mandatory when a line is being edited, null when a line has simply been added.

Comments 0 1 CommentLog

Comments for this Contract Line

AttachmentsList 1 1 AttachmentList
LinePrimaryAttachment 0 1 AttachmentKey

The primary attachment. One of the attachments above.

ItemPrimaryAttachment 0 1 AttachmentKey

The primary attachment. One of the attachments above.

Child elements

Name Minimum Maximum Type Description

The code of the company in which the contract master exists.

Code 1 1 typeCode

The contract code.

Name 1 1 typeName

ShortName 1 1 typeShortNameB

The abbreviated master name.

Shared 0 1 typeBoolean

Is this master shared?

Supplier 1 1 ElementMasterKey

The supplier for this contract. The element code.

Reference 1 1 typeDescription

Contract reference, for quoting on requisitions and orders; this is not the same as the document reference. This property is mandatory, and must be unique within a company.

Currency 1 1 CurrencyKey

The code of the currency in which the supplier trades, and which is used for the contract.

Buyer 0 1 BuyerKey

The user code of the buyer responsible for this contract. Optional; if set, must be a User Master Code which has been flagged as a buyer.

Terms 0 1 typeTermB

Contract terms, to be used in precedence to element master terms. Optional.

Discount 0 1 typePercentageb

A discount percentage to be applied to all prices within the contract, to cope with the "price list less" type contractual arrangement. Optional.

ReviewDate 0 1 typeHardb

Review date for contract. Not mandatory. If set, it must be in correct date format and greater-than-or-equal-to Start date and less-than-or-equal-to End date, if set. Cannot be in the past when modified.

ReviewUser 0 1 UserKey

The user that will receive an email to warn that the contract review date has been reached.

Description 0 1 typePopCommentTextb

Contract comments.

HeaderComments 0 1 CommentLog

Comments for this Contract Line

StartDate 1 1 typeHard

Start date for contract. Mandatory. Must be correct date format and less than or equal to End date and Review date, if either are set. Can be in the past. This must be before all other dates on the contract. End date, line start and end dates and review dates.

EndDate 0 1 typeHardb

End date for contract. Not mandatory. No entry treated as an ongoing arrangement i.e. never expires. If set, it must be in correct date format and is the last date on the contract must be after start date and line dates.

TimeStamp 1 1 short The TimeStamp number of this ContractMaster.

ModDate 0 1 dateTime The last date this ContractMaster was modified.
UsrName 0 1 string The code of the user who last modified this ContractMaster record.
