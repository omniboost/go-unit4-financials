Supplier

Represents a supplier on this RFQ - the supplier can be an element reference or an ad-hoc supplier that only exists in the context of this request.

Child elements

Name Minimum Maximum Type Description
UniqueCode 1 1 typeDbGuid

This is a GUID that will be a unique reference for the RFQ/RFQ NUMBER/SUPPLIER key combination. It is used by the supplier to uniquely identify this RFQ for them. The GUID is allocated by the server and cannot be changed.

Status 1 1 typePopRfqSupplierStatus

This is the supplier status, it defines how the supplier has interacted with us, it could be not issued, awaiting response, response received or declined.

DecisionStatus 1 1 typePopRfqDecisionStatus

This is the decision status, it defines what decision the customer has taken with regards of the prices received from the suppliers.

SupplierReason 0 1 typePopCommentTextb

If the supplier declined the RFQ proposal then enter text here to explain why.

DecisionReason 0 1 typePopCommentTextb

When a RFQ is accepted or rejected, this field stores the reason of the decision

ResponseDate 0 1 typeHardb

The date when the supplier first responded to us.

IsKnownSupplier 1 1 typeBoolean

This is TRUE if the supplier is a known element (i.e. has a valid CODA supplier element master) in which case the 'elementSupplier' field is valid. If it is FALSE then it is an unknown 'adhoc' supplier that doesn't exist - in which case the 'adhocSupplier' field is valid.

ElementSupplier 0 1 ElementMasterKey

A known supplier CODA element.

IssueRFQ. Ths issue process locks the RFQ so that updates to the core data can no longer take place. Updates to the quotes are still allowed.
