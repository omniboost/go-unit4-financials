ItemHierarchyBatchEntry

A single update to be performed in the batch. When returned from a 'batch update' XMLi call, it represents an operation that failed.

Child elements

Name Minimum Maximum Type Description
Child 0 1 ItemHierarchyRelationshipKey

The relationship ID of the child to update, if an existing node is being altered. This may also be specified when adding a new child item, to specify the relationship ID of the new hierarchy entry - useful in a batch when an Item needs to be added and then changed in one batch.

NewChildItem 0 1 ItemKey

Key to a new Item that's being added to the hierarchy.

ChildActive 0 1 typeBoolean

An update operation that may be performed. If true, the child in this relationship is set to Active. This will also update the underlying master.

If more than one update operation is specified in an ItemHierarchyBatchEntry object, just one will be arbitrarily chosen.

ChildInactive 0 1 typeBoolean

An update operation that may be performed. If true, the child in this relationship is set to Inactive. This will also update the underlying master.

If more than one update operation is specified in an ItemHierarchyBatchEntry object, just one will be arbitrarily chosen.

MoveChildToTopLevel 0 1 typeBoolean

An update operation that may be performed. If true, the child is moved to the top level in the hierarchy (it will have no parent).

If more than one update operation is specified in an ItemHierarchyBatchEntry object, just one will be arbitrarily chosen.

NewParent 0 1 ItemHierarchyRelationshipKey

An update operation that may be performed. If set, this is the relationship ID of the parent that adopts the child.

If more than one update operation is specified in an ItemHierarchyBatchEntry object, just one will be arbitrarily chosen.

NewPosition 0 1 typeULongB

An update operation that may be performed. If not null, the child is moved to this position in relation to its siblings.

This update operation can be combined with 'add item' and 'move node' operations.

DeleteChild 0 1 typeBoolean

An update operation that may be performed. If true, the child is deleted from the hierarchy. (This operation may only be applied to Item children)

If more than one update operation is specified in an ItemHierarchyBatchEntry object, just one will be arbitrarily chosen.

ChildItemCode 0 1 typeCodeB

The Item Code of the child being altered. This is not necessary for proper operation of the batch service, but is useful if this ItemHierarchyBatchEntry object is returned to the client with a WarningReason. This value will not be touched by the service; if the client needs this value to be returned with any errors, it must set the value itself before sending the batch update request.

ChildGroupCode 0 1 typeCodeB

The Group Code of the child being altered. This is not necessary for proper operation of the batch service, but is useful if this ItemHierarchyBatchEntry object is returned to the client with a WarningReason. This value will not be touched by the service; if the client needs this value to be returned with any errors, it must set the value itself before sending the batch update request.

WarningReason 0 1 typeDescription

When this object is returned from an XMLi 'batch update' call, this is the reason for failure.
