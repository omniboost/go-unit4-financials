http://www.coda.com/efinance/schemas/bankreconciliation

Root Elements

BankReconciliationRequest

This service allows you to reconcile Financials document lines with statement lines.

BankReconciliationResponse

This is the response that is returned when you reconcile Financials document lines with statement lines.

CommitRequestVerb

The CommitRequestVerb type has RequestVerb as its base type.

Commits the selected reconciliation group. When reconciling Financials document lines with an electronic statement, you cannot commit the reconciliation group until it sums to zero. When reconciling Financials document lines with a paper statement, there is no need for the reconciliation group to sum to zero.

ReconciliationParams

This element contains the parameters which can be specified when reconciling Financials document lines with statement lines.

Child elements

Name Minimum Maximum Type Description
CmpCode 1 1 typeGeneralCode The code for the company in which reconciliation is being performed.
MasterCode 1 1 typeGeneralCode The code of the reconciliation master being used for this reconciliation session.
CurrRule 0 1 typeCtHomeOrElmCur Specifies whether the reconciliation currency is the currency of the element or the currency of the posted document. When reconciling on element currency, you must specify the appropriate element level (ElmLevel). When reconciling on document currency, the lines you intend to reconcile must use the same document currency.
ElmLevel 0 1 typeElmLevel The level of the element at which the reconciliation currency is held.
CurrCode 0 1 typeGeneralCode The currency code that is used to specify which document lines are selected from the database.
FinancialsSelCode 0 1 typeGeneralCode The selector master that you want to use to retrieve the Financials document lines.
FinancialsPresCode 0 1 typeGeneralCode The presenter master that you want to use to display the Financials document lines.
StatementSelCode 0 1 typeGeneralCode The selector master that you want to use to retrieve the statement lines.
StatementPresCode 0 1 typeGeneralCode The presenter master that you want to use to display the statement lines.
Assistant 0 1 AssistantScript This element holds a script for use in Assisted Reconciliation or Background Reconciliation.
DebitAcc 0 1 typeAcCodeWild The write-off account to which debit write-offs are made.
CreditAcc 0 1 typeAcCodeWild The write-off account to which credit write-offs are made.
StatementMasterCode 0 1 typeGeneralCode The statement type code to use for this reconciliation session.
ReferenceNumberCode 0 1 typeGeneralCode The reference of the statement against which reconciliation will be run.
YearAndPeriod 1 1 typePeriod The year and period to which write-offs are posted; by default this is the current period. You must provide a year and period, even if you do not expect to make any write-offs.
Source 0 1 typeCtAssocType Specifies whether reconciliation will be run against an electronic or paper statement.
LogTitle 0 1 typeLogTitle A title for the reconciliation log. This is mandatory if LogInteractive or LogBackground are set to TRUE in the reconciliation master.
LogID 0 1 int A system-generated number identifying the log file in which this reconciliation session will be logged.
LockToken 0 1 typeText The alpha-numeric string returned by the server when you call the statement service LockRequest to lock a statement table. When you want to call a bankreconciliation service that uses the locked statement table, you must include this token to indicate that you are the user with write-access to that table. Without the LockToken, the request will fail if the statement table is marked in use by another process.
