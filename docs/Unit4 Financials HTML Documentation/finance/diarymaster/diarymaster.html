Complex Types

DiaryMaster

This element holds a diary master. Diary masters exist above company level.

Child elements

Name Minimum Maximum Type Description
Code 1 1 typeCode

The code for this diary master.

Name 1 1 typeName

The full name for this diary master.

ShortName 0 1 typeShortNameB

The abbreviated name for this diary master.

DiarySelector 0 1 typeCodeB

The code of a metadata selector master used to retrieve diary events. This selector must use the DIARY dataview and must exist in the current company (or have the 'All companies' flag set).

DiaryPresenter 0 1 typeCodeB

The code of a metadata presenter master used to display the diary event information. This presenter must use the DIARY dataview and must have the 'All companies' flag set.

BrowseTransactionsPresenter 0 1 typeCodeB

The code of a metadata presenter master used to display document details when drilling down from Your Diary into Browse Transactions. This presenter must use the DETAIL-ALL or DETAIL-BOOKS dataview and have the 'All companies' flag set. Drill down is only available if a presenter is specified here.

ToDate 1 1 typeHardSoft

The end date of the selection date range. Only diary events with action dates that fall within this date range will be retrieved.

FromDate 1 1 typeHardSoft

The start date of the selection date range. Only diary events with action dates that fall within this date range will be retrieved.

AddressCategory 0 1 typeCodeB

The address category from which the customer's telephone number will be derived. The telephone number is derived from the customer's default address unless a different category is specified here. The categories available depend upon the entity master being used by the current company for address categories. At run time, if the customer does not have an address in the chosen category, the customer's default address is used.

TimeStamp 1 1 short The TimeStamp number of this DiaryMaster.
