http://www.coda.com/efinance/schemas/auditbalance

Root Elements

AuditBalanceRequest

This service allows you to audit balance information.

AuditBalanceResponse

This is the response that is returned when you audit balance information.

ProgressStatistics

This element contains statistical information about the progress of the audit.

StartParams

This element contains the parameters used to select the balances to audit.

Complex Types

StartParams

This element contains the parameters used to select the balances to audit.

Child elements

Name Minimum Maximum Type Description
Commitments 1 1 boolean Specifies whether commitment balances will be audited.
LogTitle 1 1 typeLogTitle The title of the log that will be generated by the audit or toolbox process.
Year 1 1 short The financial year to which the documents you want to audit have been posted.
FromPeriod 1 1 typePeriods The first period in the period range from which balance information is audited.
ToPeriod 1 1 typePeriods The last period in the period range from which balance information is audited.
ElementValidation 1 1 boolean If True, the audit process must reconstruct the account code from the element codes in the balance entry and compare it with the contents of the account code column of the balance table.
RepBasisValidation 1 1 boolean If True, the audit process must reconstruct the reporting basis from the account code and compare it with the contents of the reporting basis column of the balance table.
DPValidation 1 1 boolean If True, the audit process must verify that the number of decimal places recorded on the balance entry agrees with the number recorded on the currency master for normal balances, or the balance master for quantity balances.
Toolbox 1 1 boolean Specifies whether you are running the Balance Toolbox process (which corrects any errors detected in the database).
ResetRepBases 1 1 boolean If True, resets the reporting basis.

ElapsedTime

This element contains details about the time that has elapsed in days, hours and minutes since you started the audit process.

Child elements

Name Minimum Maximum Type Description
Days 1 1 short The number of days that have elapsed since the start of the audit process.
Hours 1 1 short The number of hours that have elapsed since the start of the audit process.
Minutes 1 1 short The number of minutes that have elapsed since the start of the audit process.

ProgressStatistics

Child elements

Name Minimum Maximum Type Description

FromPeriod 1 1 typePeriods The first period in the period range from which balance information is being audited.
ToPeriod 1 1 typePeriods The last period in the period range from which balance information is being audited.
DocumentsRead 1 1 typeULong The number of documents read so far by the audit process.
ErrorCount 1 1 typeULong The number of errors detected by the audit process.
CheckedCount 1 1 typeULong The number of documents checked.
Status 1 1 typeCtAdbProgressStatus The current status of the audit process.
ElapsedTime 1 1 ElapsedTime This element contains details about the time that has elapsed in days, hours and minutes since the start of the audit process.
