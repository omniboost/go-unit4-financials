http://www.coda.com/efinance/schemas/browsedetails

Root Elements

BrowseDetailsRequest

This service allows you to browse the details of transactions in the database. (See also SelectorRequest/SelectDetails.)

BrowseDetailsResponse

This is the response that is returned when you browse document details.

Complex Types

Child elements

Name Minimum Maximum Type Description
Key 1 1 txnLineKey The key used to identify the document line.

Child elements

Name Minimum Maximum Type Description
UpdateType 1 1 typeCtUpdateId Identifies the type of document line detail you want to update, for example, comment.

ValueDiscounts 0 1 Discounts The discounts associated with the document line (update type "discount").
ValueString 0 1 string Contains update types for value strings: "description", "statuser", "bankcode" and "mediacode".
ValueDate 0 1 typeDate Contains update types for value dates: "valdate" and "duedate".
ValueUWord 0 1 typeUWord Contains update types for value uwords: a uword is an unsigned word, used for statuses like pay status or docline status.
ValuePayStatus 0 1 typeCtStatPayMultEdit The pay status, indicating how far a document has progressed through the pay process.
