Basics

This element contains parameters that determine which document lines are selected for reconciliation, and how reconciliation is performed

Child elements

Name Minimum Maximum Type Description
CurrRule 1 1 typeCtHomeOrElmCur Specifies whether the reconciliation currency is the currency of the element or the currency of the posted document.
ElmLevel 0 1 typeElmLevel The level of the element at which the reconciliation currency is held. This is a number from 1-8 inclusive.
CurrCode 1 1 typeGeneralCode The currency code that specifies the currency of document lines selected for reconciliation.
AllowInteractive 1 1 boolean Indicates whether this reconciliation master can be used for Interactive Reconciliation.
AllowBackground 1 1 boolean Indicates whether this reconciliation master can be used for Background Reconciliation.
AllowAssistant 1 1 boolean Indicates whether this reconciliation master can be used for Assisted Reconciliation. This cannot be set if AllowInteractive is set to FALSE.
LogInteractive 1 1 boolean Indicates that a log should be written during Interactive Reconciliation using this master. This can only be set when AllowInteractive is set to TRUE.
LogBackground 1 1 boolean Indicates that a log should be written during Background Reconciliation using this master. This can only be set when AllowBackground is set to TRUE.
RuntimeModifyBasics 1 1 boolean Indicates whether Basics elements can be modified at run time. When TRUE, only CurrRule, ElmLevel and CurrCode may be modified.
RuntimeModifyPosting 1 1 boolean Indicates whether Posting elements can be modified at run time. When TRUE, only DebitAcc and CreditAcc may be modified; write-off tolerances cannot be changed.
RuntimeModifyAssistant 1 1 boolean Indicates whether the Assistant script can be modified at run time.
