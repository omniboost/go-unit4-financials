http://www.coda.com/efinance/schemas/destinationmaster

Root Elements

DestinationMasterRequest

This service allows you to retrieve destination masters.

DestinationMasterResponse

This is the response that is returned when you maintain destination masters.

Complex Types

DestinationMaster

This element holds a destination master.

Child elements

Name Minimum Maximum Type Description
TimeStamp 1 1 short The TimeStamp value for this destination master in the database.
CmpCode 1 1 typeGeneralCode The code for the company in which the destination master is being maintained.
Code 1 1 typeGeneralCode The code for this destination master.
Name 1 1 typeName The full name for this destination master.
ShortName 1 1 typeShortNameB The abbreviated name for this destination master.
UseWithAllocation 1 1 boolean Specifies whether this master is used within InterCompany allocation.
TransferMethod 1 1 typeCtDesTransMethod Controls the database where the transaction will be processed.
Validation 1 1 typeCtDocPost Specifies the type of validation applied to destination document.
SubstituteAcc 1 1 typeAcCode The account code for the control account in the sending company.
ReceivingCmpCode 1 1 typeGeneralCode The code for the company that will receive the transaction.
ReceiverCode 1 1 typeGeneralCode The code for the receiver master that is used for posting the transaction.

DsmReqKeys

The DsmReqKeys type has ReqKeys as its base type.

Contains key information for requesting a number of destination masters from the database.

Child elements

Name Minimum Maximum Type Description
SelfOnly 0 1 boolean If TRUE, only destination masters of type 'Self' are returned. (If 'Self' is set the receiving company is the same as the sending company.)
