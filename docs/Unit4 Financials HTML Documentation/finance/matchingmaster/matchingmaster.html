MatchingMaster

This element holds a matching master.

This service allows you to maintain matching masters.

This is the response that is returned when you maintain matching masters.

mamReqKeys

This element is a key for requesting a number of matching masters from the database.

Child elements

Name Minimum Maximum Type Description
Key 1 1 Key This element is a key consisting of a code.
UseFromInputOnly 1 1 boolean Specifies whether only matching masters that can be used from within Input are selected.
MaxKeys 1 1 int The maximum number of matching masters to be selected. A value of zero indicates no limit.

MatchingOptions

This element contains further options to control how matching behaves.

Child elements

Name Minimum Maximum Type Description
IncSuppressPaid 1 1 boolean Specifies whether document lines marked as Suppressed Paid can be included in the selection.
CopyCommentOnPartPay 1 1 boolean Specifies that when a document line is part-paid, any comment on that document line is copied to the additional line which is posted to record the amount outstanding.
ExplodePresCode 0 1 typeGeneralCode The code of the presenter master that will be used in any exploded views of matching details.
MatchingMethod 1 1 typeCtMatchingMethod Specifies whether to perform standard or deferred matching. Standard matching selects document lines with a payment status of Available. Deferred matching selects document lines with a payment status of Draft Paid.
CopyPayData 1 1 boolean When MatchingMethod is deferred, this element specifies that all matched lines in a deferred payment process have the same matching reference and payment numbers. Also, information about banks, media and addresses will be copied to the final stage.

Name Minimum Maximum Type Description

Period 0 1 typePeriodAny The period to which documents are posted when generated by matches being committed. This can be a soft date.
DiscAcc 0 1 typeAcCodeWild The account to which discounts to the amounts due are posted.
WOffAcc 0 1 typeAcCodeWild The account to which write-offs to the amounts due are posted.
CurrWOffAcc 0 1 typeAcCodeWild The account to which currency exchange gains or losses for debits are posted. Wildcards are permitted in this account code.
RestrictAcc 1 1 boolean Specifies whether the currency write-off account code is extended by element code substitution if the * wildcard is used in the right-most position.
CredCurrWOffAcc 0 1 typeAcCodeWild The account to which currency exchange gains or losses for credits are posted. Wildcards are permitted in this account code.
CredRestrictAcc 0 1 boolean Specifies whether the currency write-off account code is extended by element code substitution if the * wildcard is used in the right-most position.
UseWriteOffMasters 1 1 boolean Specifies whether to use write-off masters in matching.

InputAndMatchingOptions

This element contains parameters that affect whether the matching master can be used from Input, and whether settlement postings are generated automatically to match off the selected lines.

Child elements

Name Minimum Maximum Type Description
UseFromInput 1 1 boolean Specifies whether this master can be used from within Input.
GenSettlementPostings 1 1 boolean Specifies whether settlement postings are generated automatically to balance off the selected lines.
SettlementDocCode 0 1 typeGeneralCode The code of the document to be used when generating settlement postings.
SuspAcc 0 1 typeAcCodeWild The account to be used for settlement postings. This account code cannot contain umbrella elements. Wildcards are permitted in this account code.
RestrictAcc 1 1 boolean Specifies whether the suspense account code is extended by element code substitution if the * wildcard is used in the right-most position.
UseCurrentRates 1 1 boolean When TRUE, the currency exchange rates applicable for the matching date are used for the settlement posting. A currency difference posting is created for the difference in currency values. When FALSE, all currency values are accumulated for the settlement posting and exchange rate differences are not possible.
EnableDataPropagation 1 1 boolean Specifies whether the data propagation functionality is enabled when Matching in Input.
Vocabs 1 1 VocabList The type of information to propagate.

MatchingMaster

This element holds a matching master.

Child elements

Name Minimum Maximum Type Description
TimeStamp 1 1 short The TimeStamp value for this matching master in the database.
CmpCode 1 1 typeGeneralCode The code for the company in which the matching master is being maintained.
Code 1 1 typeGeneralCode The code for this matching master.
Name 1 1 typeName The full name for this matching master.
ShortName 1 1 typeShortNameB The abbreviated name for this matching master.
CurrRule 1 1 typeCtMatchingCurr Specifies whether matching should be in the currency of the matchable element or in the currency of the posted document.
CurrCode 0 1 typeGeneralCode The currency code that is used to specify which document lines are selected by the search.
ElmLevel 1 1 typeElmLevel The element level at which the database is searched for document lines. It must be the matchable element level.
ElmCode 0 1 typeElmCodeWild This can contain an element code, a wildcard expression specifying a range of element codes, or can be blank if CurrCode contains a single currency code. Only documents with the specified element(s) at the specified ElmLevel are selected by the search. The code must be at the matchable element level.
TraderCode 0 1 typeTraderCodeWild This field can contain a trader code, a wildcard expression specifying a range of trader codes, or can be blank if you have not specified an umbrella element code in ElmCode. If the matchable element is an umbrella element you must specify a trader code because matching can only take place between documents with the same temporary supplier.
DiscDate 0 1 typeHSDate The date used to determine which discounts are processed. This can be a soft date.

Folders 1 1 Folders This element contains parameters to determine which document lines are selected for matching and how they are presented.
Assistant 1 1 AssistantScript This element holds a script for use in Assisted Matching or Background Matching.
