http://www.coda.com/efinance/schemas/browsereportmaster

Root Elements

BrowseReportMasterRequest

This service allows you to maintain browse report masters.

BrowseReportMasterResponse

This is the response that is returned when you maintain browse report masters.

Complex Types

BrowseReportMaster

This element holds information for a browse report master. This information is used to retrieve the Browse Balances results from the database.

Child elements

Name Minimum Maximum Type Description
CmpCode 1 1 typeGeneralCode The code for the company in which the browse report master is being maintained.
TimeStamp 1 1 short The TimeStamp value for this browse report master in the database.
Code 1 1 typeGeneralCode The code for the browse report master.
Name 1 1 typeName The full name for the browse report master.
ShortName 1 1 typeShortNameB The abbreviated name for the browse report master.
ReportType 1 1 typeCtBrpReportType Indicates the type of browse report master you are creating or maintaining. The available values are: Document, Detail, Balance, Ledger and Assets.
BrowseBalMasterCode 0 1 typeGeneralCode The code for the browse balance master you want to use.
SelMasterCode 0 1 typeGeneralCode The code for the selector master you want to use.
ReportingBasis 0 1 short The reporting basis you want to use. Each element level is represented by a check box which you can set. The reporting basis defined here will override any reporting base criteria defined on the selector master.
IncludeIntrayDetails 0 1 boolean Indicates whether documents located on the Financials Intray are included when you drill to Browse Details.
EnableDrillToDetails 0 1 boolean Indicates whether you can drill down from Browse Balances to Browse Details. Setting this check box may increase the time taken to retrieve the balance information from the database.
ExcludeZeroBalances 0 1 boolean Indicates whether balances with a value of zero are excluded from the display.
DisableViewPeriods 0 1 boolean Indicates whether you can use the View Periods functionality with the data that is returned by this report. If you select this check box, the search of the database will be faster than otherwise because the additional information required for period explode will not be selected.
DetailsSelector 0 1 typeGeneralCodeB The code for the selector master you want to use for a Browse Details report.
DetailsPresenter 0 1 typeGeneralCodeB The code for the presenter master you want to use for a Browse Details report.
DetailsExplode 0 1 typeGeneralCodeB The code for the presenter master you want to use when exploding a document in Browse Details.
Ledger 0 1 typeGeneralCodeB The code for the ledger master you want to use for a Browse Ledger report.
AssetSelector 0 1 typeGeneralCodeB The code for the selector master you want to use for a Browse Assets report.
AssetPresenter 0 1 typeGeneralCodeB The code for the presenter master you want to use for a Browse Assets report.
SourcePresenter 0 1 typeGeneralCodeB The code for the presenter master used in a Browse Assets report to display data in the Source Transactions form.
IncludeChildren 0 1 boolean Specifies whether children of the selected assets are included, irrespective of whether they are included in the selection themselves. To display child assets together with their parent asset, you must use the sort order functionality.
DisplayBook 0 1 typeGeneralCodeB Includes book level data, such as Net Book Value, in the list view. The posting book for the company is selected by default but you may select another book. An option for All Books will display a line for each asset and book combination.
HeaderAndFooterOccurrence 0 1 typeBrowseBalanceHFOccurrence Indicates when the header and footer occurs on the report. The available values are: Never, On each report page and At the start and end of the report.
DisplaySelectionCriteria 1 1 typeCtBrpDisplaySelCriteriaLocation Specifies whether the selection criteria are displayed at the beginning of the report.
HeaderHasEqualWidthColumns 0 1 boolean Indicates whether the columns in the report header are equal widths.
FooterHasEqualWidthColumns 0 1 boolean Indicates whether the columns in the report footer are equal widths.
RowsPerPage 0 1 short Specifies the maximum number of rows that are printed per page for a Browse Balances report.
SuppressZeroBalances 0 1 boolean If true, balances which old zero values are not posted to Financials.
Comment 1 1 typeTextAnyB The text of the comment.
CommentLocation 1 1 typeCtBrpCommentLocation Specifies where you want the comment to appear.
CommentExport 1 1 boolean Specifies that you want to include the comment when the report is exported.
MatchingMasterCode 1 1 typeGeneralCodeB The code of the default matching master held on the browse report master, used when matching from within Browse Details.
HideRepeats 1 1 boolean Specifies that textual (account) information is not displayed when the value would be the same as in the row above.
MatchMasFixedDefault 1 1 boolean Specifies whether the default matching master is fixed (that is, cannot be overridden at run time).
InputTemplateMasterCode 1 1 typeGeneralCodeB The default input template to use when raising a corrective journal.
InputTemplateCodeFixedDefault 1 1 boolean Specifies that this input template must be used when you raise a corrective journal and cannot be overridden at run time.
PromptRefYearPeriod 0 1 boolean Specifies whether the user is prompted for the reference year/period on the Run Browse Balances Report and Run Browse Report forms.
RefYearPeriod 0 1 typePeriod The reference year/period that is used to resolve soft year/periods. If left blank, the current year/period (when the report is run) is used as the reference year/period.
ReportHeader 1 1 HeaderFooterDefinition This element contains the column information for the report header.
ReportFooter 1 1 HeaderFooterDefinition This element contains the column information for the report footer.
CorrectiveBehaviour 0 1 typeCtBrpCorrectiveBehaviour Specifies whether the senses of the original lines are preserved or reversed on the corresponding generated lines in a corrective journal.
CorrectiveBehaviourFixedDefault 0 1 boolean If true, specifies that the above setting for whether the senses of the original lines are preserved or reversed must be used and cannot be overridden at run time.
