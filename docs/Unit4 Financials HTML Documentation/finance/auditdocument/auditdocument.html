http://www.coda.com/efinance/schemas/auditdocument

Root Elements

AuditDocumentRequest

This service allows you to check and verify that the documents in your Financials database are in balance in document, dual, home and any self-balancing currencies.

AuditDocumentResponse

This is the response that is returned when you audit documents.

ProgressStatistics

This element contains statistical information about the progress of the audit.

StartParams

This element contains the parameters used to select the documents to audit.

Complex Types

StartParams

This element contains the parameters used to select the documents to audit.

Child elements

Name Minimum Maximum Type Description
LogTitle 1 1 typeLogTitle The title of the log that will be generated by the audit process.
SelCode 0 1 typeGeneralCode The code for the selector master used to limit the number of documents selected for auditing. If you leave this field blank all the documents in the specified year and period range are audited.
SelectorVocabList 0 1 VocListData This element holds the selection criteria that determine which documents are selected for inclusion in the audit process.
Year 1 1 short The financial year to which the documents you want to audit have been posted.
FromPeriod 1 1 typePeriods The first period in the period range to which the documents have been posted.
ToPeriod 1 1 typePeriods The last period in the period range to which the documents have been posted.
CheckMissingLines 1 1 boolean If True, the audit process will check that headers have associated document lines and currency lines. If False, the process will run faster.

ProgressStatistics

This element contains statistical information about the progress of the audit.

Child elements

Name Minimum Maximum Type Description
Year 1 1 short The financial year being audited.
FromPeriod 1 1 typePeriods The first period in the period range from which documents are being audited.
ToPeriod 1 1 typePeriods The last period in the period range from which documents are being audited.
DocumentLines 1 1 typeULong The number of document lines read so far by the audit process.
ErrorCount 1 1 typeULong The number of errors detected by the audit process.
Status 1 1 typeCtAddProgressStatus The current status of the audit process.
ElapsedTime 1 1 ElapsedTime This element contains details about the time that has elapsed in days, hours and minutes since the start of the audit process.
