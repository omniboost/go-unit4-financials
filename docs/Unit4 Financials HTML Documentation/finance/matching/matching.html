Root Elements

GroupResults

This element contains the results of committing or dispersing multiple matches.

MatchGroups

This element contains details of the matching groups in the matching session.

MatchingParams

This element contains the parameters which can be specified when matching debit and credit entries.

MatchingRequest

This service allows you to match outstanding debit and credit entries.

MatchingResponse

This is the response that is returned when you match debit and credit entries.

UserRefs

This element contains default user references that will be copied to all document lines in the current matching session.

MatchingParams

This element contains the parameters which can be specified when matching debit and credit entries.

Child elements

Name Minimum Maximum Type Description
CmpCode 1 1 typeGeneralCode The code for the company in which matching is being performed.
MatchingMasterCode 1 1 typeGeneralCode The code of the matching master being used for this matching session.
ElmLevel 0 1 typeElmLevel The element level at which the database is searched for document lines. It must be the matchable element level.
ElmCode 0 1 typeElmCodeWild The code of the matchable element whose debit/credit entries you want to match.
TraderCode 0 1 typeTraderCodeWildB The code of a trader (umbrella element) whose debit/credit entries you want to match.
MatchingDate 1 1 typeDate The date on which matching is performed. This is used for the document date if a match is committed.
Period 0 1 typePeriod The period to which documents are posted when generated by matches being committed.
DiscDate 0 1 typeDate The date used to determine which discounts are processed.
CurrRule 0 1 typeCtMatchingCurr Specifies whether matching should be in the currency of the matchable element or in the document currency. When matching on document currency, the documents you intend to match must use the same document currency.
CurrCode 0 1 typeGeneralCodeB The currency code that is used to specify which document lines are selected from the database.
LogTitle 0 1 typeLogTitleB An optional log title. All attempts to commit during the matching session are stored under this log title.
LogErrorOnly 0 1 boolean Indicates whether only errors are logged, as opposed to both errors and successful commits. When TRUE, performance can be improved significantly, particularly with Background Matching processes.
LogID 0 1 int A system-generated number identifying the log file in which this matching session will be logged.
RowLimit 0 1 typeULong The maximum number of lines that can be listed in each matching folder.
MatchingFromBrowse 0 1 boolean If true, indicates that matching has ben initiated from Browse.
