http://www.coda.com/efinance/schemas/input

Root Elements

InputRequest

This service allows you to post financial documents to the Books or the Intray.

InputResponse

This is the response that is returned when you post financial documents to the Books or the Intray.

Complex Types

PostRequest

The PostRequest type has Request as its base type.

Child elements

Name Minimum Maximum Type Description
Transaction 1 1 Transaction Contains the document you want to post.

PostRequestVerb

The PostRequestVerb type has RequestVerb as its base type.

Posts the document.

Attributes

Name Type Use Default Description
logerrorsonly boolean optional If True, specifies that the log contains only errors.
postto typeCtDocDest optional Specifies whether the document should be posted to the Books, the Intray, or either (preferentially the Books but failing that, the Intray).
relaxtriangulation boolean optional If True, specifies that the EMU Triangulation rules do not apply to this document.
reportallerrors boolean optional If True, specifies that the process should report all errors in the log. If False, only the first error encountered under this verb is reported.
stoponerror boolean optional If True, specifies that the process is to terminate on encountering an error.
warningsareerrors boolean optional If True, specifies that warnings are to be treated as errors.

Child elements

Name Minimum Maximum Type Description
Request 1 unbounded PostRequest
