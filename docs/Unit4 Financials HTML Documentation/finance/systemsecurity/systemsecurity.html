SystemSecurity

This document holds the system security parameters: how passwords are controlled, which logon details you want to monitor, and which users are locked out.

Child elements

Name Minimum Maximum Type Description
TimeStamp 1 1 short The TimeStamp for the System Security master in the database.
MinPasswordLen 1 1 short The minimum length of the password, between 4 and 12.
MinAlphaChars 1 1 short The minimum number of alphabetic characters (A-Z, a-z) that must be included in a password. The minimum value is 0 and the maximum is 12.
MinNumericChars 1 1 short The minimum number of digits that must be included in a password. The minimum value is 0 and the maximum is 12.
MinOtherChars 1 1 short The minimum number of 'other' characters. The minimum value is 0 and the maximum is 12.
ProhibitOldPasswords 1 1 boolean Specifies that users cannot reuse their old passwords.
OldPasswordListLength 1 1 short The size of the list of previous passwords that any new password must be checked against before it can be used.
UseTimeLimits 1 1 boolean If TRUE, specifies that logon passwords will have a specific life span.
PasswordLife 1 1 short The number of days that logon passwords will be usable before the users have to change them.
PasswordExpiryWarning 1 1 short The number of days warning users will be given that the password is about to expire.
LimitConsecutiveFailures 1 1 boolean If TRUE, specifies that you want to limit the number of attempts at logging on a user will be allowed before being locked out.
MaxLogonFailures 1 1 short The number of attempts at logging on users will be allowed.
RecordSuccessfulLogons 1 1 boolean If TRUE, specifies that you want to record all successful logons and logoffs to the audit trail table. If untrue, only unsuccessful logon attempts are recorded.
ChangeOnFirstUse 1 1 boolean If TRUE, specifies that a user must change his password the first time he logs on to the application.
