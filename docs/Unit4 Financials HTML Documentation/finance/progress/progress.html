http://www.coda.com/efinance/schemas/progress

Root Elements

ProgressRequest

This service allows you to request information from the server about a long-running process, or to abort it. It is used by several processes, including Archive, Tax Transfer, Allocation, and Pay.

ProgressResponse

This is the response that is returned when you use the GetProgress or Abort request for a long-running process on the server.

ProgressRequest

The ProgressRequest type has ServiceRequest as its base type.

This service allows you to request information from the server about a long-running process, or to abort it. It is used by several processes, including Archive, Tax Transfer, Allocation, and Pay.

Child elements

Name Minimum Maximum Type Description
GetProgress 0 1 progressGetRequestVerb Retrieves information about the progress of a long-running process on the server.

ProgressKey

Contains the key, or special reference, given to you for monitoring a long-running process on the server. You will be supplied with one of the following keys, depending on the service you are using.

Child elements

Name Minimum Maximum Type Description
Key 0 1 typeULong A unique key which identifies the process.
CodeKey 0 1 Key An alternative key for identifying the process, which is used by Archive, for example.

ProgressData

Contains information about the progress of the Financials process you are monitoring.

Child elements

Name Minimum Maximum Type Description
Message 1 1 typeProgressMessage Contains information about the progress of this process.
Status 1 1 typeCtProgressStatus The current status of the process. This may be Completed successfully, Completed with errors, In progress, or Aborted.
StepsProcessed 0 1 int The number of steps completed so far.
StepsToProcess 0 1 int The number of steps not yet completed.
