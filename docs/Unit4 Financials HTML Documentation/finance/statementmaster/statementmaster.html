Root Elements

StatementMaster

This element holds a statement type master.

StatementMasterRequest

This service allows you to maintain statement type masters.

StatementMasterResponse

This is the response that is returned when you maintain statement type masters.

TransactionMaster

This element holds a transaction type master.

TransactionMasterRequest

This service allows you to maintain transaction type masters.

TransactionMasterResponse

This is the response that is returned when you maintain transaction type masters.

Name 1 1 typeName The full name of the master.
ShortName 0 1 typeShortName The abbreviated name of the master.
PreCode 1 1 typeGeneralCode The code of the default presenter you want this statement type master to use.
Balancing 1 1 boolean If TRUE, specifies that statements of this type are self-balancing.

TransactionMaster

This element holds a transaction type master.

Child elements

Name Minimum Maximum Type Description
CmpCode 1 1 typeGeneralCode The code of the company in which the transaction type master exists.
Code 1 1 typeGeneralCode The code for this transaction type master.
TimeStamp 0 1 short The TimeStamp value for the transaction type master in the database.
Name 1 1 typeName The full name of the master.
ShortName 0 1 typeShortName The abbreviated name of the master.
StmCode 1 1 typeGeneralCode The code of the statement type master that you want to use with this transaction type.
ExternalCode 1 1 typeGeneralCode The transaction identifier that appears in the statement for lines of this type.
DocSrcType 1 1 typeCtDocumType Where the document for posting the statement lines is derived from: Specify, or From file.
DocCode 0 1 typeGeneralCode The code of the document used to post the statement lines, if the DocSrcType is Specify.
DocNumType 1 1 typeCtDocumentNumbering How often a separate document is created, determining the number of documents: Document per item, Document per process, or From file.
Location 1 1 typePostingLocation The location to which the document will be posted: Books or Intray.
Reconcile 1 1 boolean If TRUE, specifies that lines of this transaction type may be automatically reconciled after posting.
ExtensionDLL 1 1 typeExtDLL The full path for the DLL file you want to use.
SrcAccSrc 1 1 typeCtAcccodeSource How the account code to which the source lines are posted is derived: Specified, Lookup Account or From File.
SrcAccCode 1 1 typeAcCode The code of the account to which source lines of this type are posted, if you have selected Specified in SrcAccSrc.
SrcDesc 0 1 typeDescr The description to be used instead of any description that is in the statement file. You must enter at least one printable character.
SrcLineType 1 1 typeCtDoclinetypeNotax The line type of the source lines: Summary, Analysis or taken From file.
SrcCheckExtRefs 1 1 boolean If TRUE, specifies that external references are checked for source lines of this type when you validate a statement. The checking rules specified on the document master are used.
SrcValidateAcc 1 1 boolean If TRUE, specifies that the account code is validated for source lines of this type when you validate a statement. The standard Financials validation is used.
SrcExtValidation 1 1 boolean Specifies that the user extension named in UserExtension is called for source lines of this type when you validate a statement. This user extension can check and modify any of the external references.
ContraLineSrc 1 1 typeCtContraSource How the value of the contra line is derived: Calculated or From file.
ContraAccSrc 1 1 typeCtAcccodeContra How the account code to which contra lines are posted is derived: Specified, Lookup account, or From file.
ContraAccCode 1 1 typeAcCode The code of the account to which contra lines of this type are posted, if you have selected Specified in ContraAccSrc.
ContraDesc 0 1 typeDescr The description to be used instead of any description that is in the statement file. You must enter at least one printable character.
CopyExtRefList 0 1 CopyExtRefList The list of external references which you want to copy.
ContraLineType 1 1 typeCtDoclinetypeNotax The line type of the contra lines: Summary, Analysis or taken From file.
ContraCheckExtRefs 1 1 boolean Specifies that external references are checked for contra lines of this type when you validate a statement. The checking rules specified in the document master are used.
ContraValidateAcc 1 1 boolean Specifies that the account code is validated for contra lines of this type when you validate a statement. The standard Financials validation is used.
ContraExtValidation 1 1 boolean Specifies that the user extension master named in UserExtension is called for contra lines of this type when you validate a statement. This implementation can check and modify any of the external references.
RelaxTriangulation 1 1 boolean If TRUE, specifies that the EMU triangulation rules for currency conversion do not apply to the calculation of the contra line values.
UserExtension 0 1 ExtensionRef The code of the extension master you want to use for validation.
