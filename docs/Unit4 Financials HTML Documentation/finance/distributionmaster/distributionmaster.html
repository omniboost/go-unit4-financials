http://www.coda.com/efinance/schemas/distributionmaster

Root Elements

DistributionMaster

This document holds a distribution master.

DistributionMasterRequest

This service allows you to maintain distribution masters.

DistributionMasterResponse

This is the response that is returned when you maintain distribution masters.

Complex Types

Master

Child elements

Name Minimum Maximum Type Description
CmpCode 1 1 typeGeneralCodeB The code of the company in which this distribution master exists.
Code 1 1 typeGeneralCode The code for this distribution master.
TimeStamp 1 1 short The TimeStamp value for this distribution master in the database.
Name 1 1 typeName The full name of the distribution master.
ShortName 1 1 typeShortNameB The abbreviated name of the distribution master.
Balance 1 1 typeGeneralCodeB The master code that specifies the balances to which you want to distribute the selected values.
Currency 1 1 typeGeneralCodeB The currency in which the balances are to be updated.
DistributionType 1 1 typeCtDstDistributionType The type of distribution this master uses: proportional or basis.
BasisCurrency 1 1 typeCtDstBasisCurrency For basis distribution, whether allocation is made on the basis of selected home or dual currency values.
UseWithInterCompany 1 1 boolean If TRUE, specifies that this master is used for InterCompany Allocation.
DistributeToBalances 1 1 boolean If TRUE, specifies that distribution is to balances.

Distribution

An item on the distribution list.

Child elements

Name Minimum Maximum Type Description
AccountCode 1 1 typeAcCode The account to which part of the cost will be allocated.
Weight 1 1 typeNum For proportional distribution, the proportion of the total that you want to allocate to the named account.
ReportingBasis 1 1 typeRepBasis If distribution is to balances, the reporting basis which indicates the intersects at which the balance is held. Zero represents the full account code.
SelectorCode 1 1 typeGeneralCode The selector code which determines the values that you want to allocate to the named account in basis distribution.
DestinationCode 1 1 typeGeneralCodeB The code of the destination master holding the information about the destination company, if you are distributing across companies.
ElementSelectorList 1 1 ElementSelectorList The list of elements requiring resolution of a wildcard in the account code (wildcards can be entered for basis distribution only).

DistributionList

The list of accounts to which the values are allocated.

Child elements

Name Minimum Maximum Type Description
Distribution 1 unbounded Distribution An item on the distribution list.

ElementSelectorList

The list of elements requiring resolution of a wildcard in the account code (wildcards can be entered for basis distribution only).

Child elements

Name Minimum Maximum Type Description
ElementSelector 8 8 typeCtElementSelectorType How the wildcard will be resolved: by reference to the basis accounts (basis) or by reference to the selection accounts (rule) .

Key

A distribution master key.

Child elements

Name Minimum Maximum Type Description
CmpCode 1 1 typeGeneralCode The code of the company in which the distribution master exists.
Code 1 1 typeGeneralCode The code of the distribution master.
Type 0 1 typeCtDstDistributionType The type of distribution master: proportional or basis. If you leave this field empty, all keys are returned.
UseWithInterCompany 0 1 boolean If TRUE, this distribution master can be used with InterCompany (relevant for Allocation only).
DistributeToBalances 0 1 boolean If TRUE, specifies that distribution is to Balances. Otherwise distribution is to Details.

dstReqKeys

Contains key information for requesting a number of distribution masters from the database. Ommission implies you want to fetch ALL Keys rather than fetching keys by their type (basis or proportional)

WildKey 1 1 Key A distribution master key.
ShortName 1 1 typeShortNameB The short name of the distribution master.
