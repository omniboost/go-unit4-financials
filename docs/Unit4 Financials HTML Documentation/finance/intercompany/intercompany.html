http://www.coda.com/efinance/schemas/intercompany

Root Elements

InterCompanyRequest

This service allows you to process InterCompany documents.

InterCompanyResponse

This is the response that is returned when you process the InterCompany documents.

Complex Types

OverrideData

Specifies that you can override some of the values supplied by the designated receiver master with your own values.

Child elements

Name Minimum Maximum Type Description
ReceivingDocCode 1 1 typeGeneralCode The code of the document master to be used to post details in the receiving company.
InitialLocation 1 1 typeCtRecInitialLoc Specifies where in the receiving company the receiving process attempts to post document lines retrieved from the InterCompany Trace.
YearPeriod 1 1 typePeriod This year and period overrides the year and period calculated using the rule on the receiver master. The values entered must be a valid year and period for the receiving company.
ForwardDestination 0 1 typeGeneralCodeB The destination code for the company to which the document line is to be forwarded. If this code is left blank, any forward destination on the receiver master is overridden and the retrieved document lines are not forwarded, but posted in the current receiving company.

ReceiverData

This element contains information about the receiver process.

Child elements

Name Minimum Maximum Type Description
GslCode 1 1 typeGeneralCodeB The code for the selector master used to retrieve document lines from the InterCompany Trace.
TreatWarningsAsErrors 1 1 boolean Specifies that a warning is treated as an error.
StopRunOnErrors 1 1 boolean Stops the InterCompany process when an error is encountered.
ContinueForwarding 1 1 boolean Specifies that InterCompany forwarding continues automatically through the intermediate companies until the selector can no longer select any transactions from the InterCompany Trace.
AccumulateLines 1 1 boolean Specifies that lines from the same original document that are posted to the same company are included in the same posting document.
AccumulationLimit 1 1 int The maximum number of accumulated lines in a document. The maximum you can specify is 999,999. Any lines above the specified limit are accumulated into another document. A limit of 0 means that all lines are accumulated into one document, not that no lines are accumulated.
Overrides 0 1 OverrideData Specifies that you can override some of the values supplied by the designated receiver master with your own values.
Selection 1 1 VocListData The selection criteria that determines which document lines are selected for inclusion in the receiver process.

BasicProcessData

This element contains information about the basic InterCompany processes.

Child elements

Name Minimum Maximum Type Description
LogTitle 1 1 typeLogTitle The title of the log that will be generated by the process.

ProcessData

This element contains data about the two InterCompany processes: the Receiver process and the Cross-database process.

Child elements

Name Minimum Maximum Type Description
BasicProcessData 1 1 BasicProcessData This element contains information about the basic InterCompany processes.
ReceiverData 1 1 ReceiverData This element contains information about the receiver process.
