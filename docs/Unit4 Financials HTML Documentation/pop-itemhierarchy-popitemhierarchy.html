POPItemHierarchyService

Namespace: http://www.coda.com/pop/schemas/itemhierarchy/popitemhierarchy-12.0/webservice

Path: services/pop/itemhierarchy/popitemhierarchy-12.0

The following methods are available on this endpoint:

  • BatchUpdate
  • DeleteItem
  • GetChildren
  • GetFilteredChildren
  • GetParents
  • Move
  • SetActive
  • SetChildPosition

AddItem

Input Parameters

AddItemRequest
Name Minimum Maximum Type Description
Item 1 1 ItemKey

Key of the Item to add to the hierarchy.

ParentProductGroupMaster 0 1 ProductGroupMasterKey

Key of the Product Group Master which will adopt the child. If this is null then the Item is added at the top level in the hierarchy.

Position 1 1 typeULong

The position of the new Item in relation to its siblings. If zero (or negative), then this item will head the list. If greater than the number of items in the list, it will be appended to the end (and given a new appropriate number). If the specified position is already occupied then it will be inserted at that position.

Return Value

AddItemResponse
Name Minimum Maximum Type Description
AddItem 1 1 ItemEntryList The return value of the addItem function.

Move

Input Parameters

MoveRequest
Name Minimum Maximum Type Description
HierarchyEntry 1 1 ItemHierarchyRelationshipKey

The entry in the hierarchy to move.

ParentRelationship 0 1 ItemHierarchyRelationshipKey

Key of the ItemHierarchyRelationship which will adopt the child. This relationship must represent a ProductGroupMaster child. If this and the 'parentProductGroupMaster' is null then the Item is added at the top level in the hierarchy. This takes precedence over 'parentProductGroupMaster' if values for both are supplied.

ParentProductGroupMaster 0 1 ProductGroupMasterKey

Key of the Product Group Master which will adopt the child. If this and the 'parentRelationship' is null then Item is added at the top level in the hierarchy.

Position 0 1 typeULongB

The position of the entry in relation to its siblings. If zero (or negative), then this item will head the list. If null, it will be appended to the end (and given an appropriate number). If the specified position is already occupied then it will be inserted at that position.

Return Value

MoveResponse
Name Minimum Maximum Type Description
Move 1 1 ItemEntryList The return value of the move function.
